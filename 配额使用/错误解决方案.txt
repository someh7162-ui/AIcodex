情况 A: Node.js 版本过低
─────────────────────────────────────────────────────

  错误信息:
    [ERROR] Node.js version must be >= 18.0.0

  解决方案:
    1. 访问: https://nodejs.org/
    2. 下载 LTS 版本（左边的按钮）
    3. 安装（建议卸载旧版本）
    4. 重启电脑
    5. 验证: 打开 CMD 运行 node --version

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

情况 B: 网络连接问题
─────────────────────────────────────────────────────

  错误信息:
    [ERROR] Basic fetch failed
    Error: connect ECONNREFUSED
    或
    Error: getaddrinfo ENOTFOUND

  可能原因:
    □ 没有网络连接
    □ 防火墙阻止
    □ 需要配置代理
    □ DNS 问题

  解决步骤:

  1. 检查网络连接
     打开浏览器访问: https://www.google.com
     能打开 → 继续下一步
     不能打开 → 修复网络连接

  2. 检查是否需要代理
     
     如果公司网络需要代理，创建文件 run-with-proxy.bat:

     @echo off
     set HTTP_PROXY=http://代理地址:端口
     set HTTPS_PROXY=http://代理地址:端口
     node antigravity-quota-checker.js
     pause

     替换 "代理地址:端口" 为实际值，例如:
     set HTTP_PROXY=http://proxy.company.com:8080

  3. 尝试禁用防火墙（临时）
     控制面板 → Windows Defender 防火墙 → 关闭
     测试是否能运行
     如果可以 → 添加 Node.js 到防火墙白名单

  4. 检查 DNS
     打开 CMD 运行:
     ping oauth2.googleapis.com
     ping cloudcode-pa.googleapis.com

     如果失败，尝试更换 DNS:
     - 使用 Google DNS: 8.8.8.8
     - 使用 Cloudflare DNS: 1.1.1.1

情况 C: SSL/证书问题
─────────────────────────────────────────────────────

  错误信息:
    Error: certificate has expired
    或
    Error: self signed certificate

  临时解决方案（仅用于测试）:

  创建文件 run-skip-ssl.bat:

  @echo off
  set NODE_TLS_REJECT_UNAUTHORIZED=0
  node antigravity-quota-checker.js
  pause

  ⚠️ 警告: 这会跳过 SSL 验证，仅用于诊断

  永久解决方案:
  - 更新系统证书
  - 检查系统时间是否正确
  - 如果是公司代理，安装公司证书

情况 D: OAuth Token 过期
─────────────────────────────────────────────────────

  错误信息:
    Token refresh failed
    Error: invalid_grant

  解决方案:
    
    打开 CMD 运行:
    opencode auth login

    重新登录 Google 账户

【快速参考】常见错误代码
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ETIMEDOUT        → 连接超时
  ECONNRESET       → 连接被重置
  certificate      → SSL 证书问题
  invalid_grant    → OAuth token 过期
